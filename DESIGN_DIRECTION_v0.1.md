# StepCafeGacha デザイン方向 v0.1

## 1. 推奨タイトル

第一候補: Walk Cafe Days

日本語表記案: ウォークカフェデイズ

理由:

- 歩数連動であることが自然に伝わる。
- カフェ経営ゲームとして柔らかい印象がある。
- ガチャや放置要素を後から足してもタイトルが狭くなりすぎない。
- iOSアプリ名として短く、覚えやすい。

## 2. タイトル候補

| 優先 | タイトル | 印象 |
| ---: | --- | --- |
| 1 | Walk Cafe Days | 歩く、日常、カフェ育成が伝わる本命 |
| 2 | Step Cafe Story | 歩数と物語性を出しやすい |
| 3 | Cafe Stepper | カジュアルでゲーム名らしい |
| 4 | Walk & Brew | コーヒー感が強い。英語圏向けにも自然 |
| 5 | あるくカフェ日和 | 日本語で親しみやすい |
| 6 | 歩いてカフェ経営 | 内容は明快だが説明的 |

MVPではコード名 `StepCafeGacha` のまま進め、アプリ表示名は `Walk Cafe Days` を第一候補にする。

## 3. ビジュアル方向

推奨: かわいい現代カフェ + 少しだけファンタジー感

方向性:

- ベースは現代の小さなカフェ。
- 木目、白い壁、観葉植物、手書き看板、焼き菓子、ラテアートを中心にする。
- ファンタジー要素は控えめに、SSRスタッフや高ランク店舗の演出にだけ入れる。
- 画面全体は明るく、清潔で、朝から昼の空気感にする。
- ガチャ演出は派手すぎず、カフェのベル、光るメニューカード、コーヒーの湯気などで表現する。

避ける方向:

- 重いRPG風UI
- 暗い酒場・ギルド風
- 過度な異世界ファンタジー
- 子供向けすぎる原色UI
- 一色だけに寄った配色

## 4. カラーパレット

主色:

- クリームホワイト: 背景、カード面
- コーヒーブラウン: 見出し、強調線
- ミントグリーン: 成長、歩数、達成報酬
- ストロベリーピンク: ガチャ、レア演出、スイーツ
- ゴールド: SSR、報酬、店舗ランクアップ

注意:

- ベージュ・ブラウンだけに寄せすぎない。
- ミント、ピンク、ゴールドを差し色にして、カフェらしさとゲームらしさを両立する。

## 5. キャラクター絵柄

推奨:

- 2Dイラスト調。
- 等身は高すぎず、3.5から5頭身程度。
- 表情が読みやすい。
- 現代カフェ制服をベースに、レアリティごとに小物や装飾を増やす。
- 背景透過PNG。
- スプライトシート禁止。キャラごとに個別PNGで管理する。

レアリティ差:

| レア | 見た目 |
| --- | --- |
| N | シンプルなエプロン、控えめな色 |
| R | 役割が分かる小物を追加。バリスタ器具、トレイなど |
| SR | 差し色、髪飾り、専用衣装感を追加 |
| SSR | 看板キャラ級。背景なしでも映えるポーズと装飾 |

## 6. 店舗ビジュアル

店舗ランクごとの見た目:

| ランク | 外観・内装 |
| --- | --- |
| 1 | 小さな町角カフェ。木製カウンター、2席、手書き看板 |
| 2 | 焼き菓子ケースと観葉植物が増える |
| 3 | テラス席、明るい窓、常連客の雰囲気 |
| 4 | スイーツのショーケース、広い客席、季節装飾 |
| 5 | 街の人気店。大きな看板、行列、特別な内装 |

## 7. UI方向

- iPhone縦画面前提。
- ホームはカフェの様子を中心に見せる。
- 数値は上部HUDにまとめる。
- 下部タブは `ホーム / 歩数 / ガチャ / 編成 / 強化` を基本にする。
- ガチャ結果はカード型で表示する。
- 編成画面はスタッフ、レシピ、設備を横断して比較しやすくする。
- テキスト量は控えめにし、数値・アイコン・カードで状態を理解できるようにする。

## 8. 初期アセット方針

最初に必要なChatGPT生成画像:

1. カフェ店内背景 ランク1
2. カフェ外観 ランク1
3. 初期スタッフ N ミナ 背景透過PNG
4. 初期スタッフ N ソウタ 背景透過PNG
5. R ハル 背景透過PNG
6. R リコ 背景透過PNG
7. ガチャ結果カード背景
8. 歩数報酬アイコン一式

画像生成ルール:

- ChatGPT生成画像を使用する。
- キャラクターは背景透過PNG。
- 透過が難しい場合は単色クロマキー背景で生成して、後処理で透過する。
- スプライトシートは禁止。
- UIに文字を焼き込まない。
