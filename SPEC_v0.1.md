# StepCafeGacha 仕様書 v0.1

## 1. 企画概要

タイトル仮称: Step Cafe Gacha

ジャンル: 歩数連動 / 放置カフェ経営 / キャラ・レシピ・設備ガチャ / 収集育成

プレイヤーは小さな現代カフェのオーナー。現実で歩くことでガチャチケット、育成素材、営業ブーストを得る。ガチャでスタッフ、レシピ、設備を集め、編成と育成によってカフェの売上・評判・来客数を伸ばしていく。

MVPではリアルマネー課金なし。ガチャは歩数報酬とゲーム内報酬で回す無料ガチャとして扱う。

## 2. 制作ルール

- 今後の作業場所は `D:\Codex\StepCafeGacha` を優先する。
- 変更作業前にバックアップを取得する。
- 画像素材は必ずChatGPT生成素材を使用する。
- キャラクター画像は背景透過PNGで使用する。
- スプライトシートは禁止。必要な場合は事前確認する。
- iOSアプリはSwiftUIを基本にする。
- 歩数取得はHealthKitを第一候補にする。
- Windows上では実機ビルド不可のため、必要に応じてWebプレビューやモックで確認する。

## 3. コア体験

歩くほどカフェ経営が進むが、歩数だけで完結しない。ゲームとしての主軸は「集める」「編成する」「育てる」「営業結果を見る」「次の強化方針を決める」こと。

基本ループ:

1. 現実で歩く
2. 今日の歩数をアプリで確認する
3. 歩数報酬を受け取る
4. ガチャチケット・育成素材・営業ブーストを得る
5. スタッフ・レシピ・設備をガチャで入手する
6. 編成してカフェ営業力を上げる
7. 放置営業でコイン・評判・経験値を得る
8. スタッフ、レシピ、設備、店舗ランクを強化する
9. 新しい客層、メニュー、設備枠、ガチャ内容が解放される

## 5. 歩数報酬

歩数は直接コイン化しすぎず、ガチャ・育成・営業ブーストの入口にする。

日次報酬案:

| 歩数 | 報酬 |
| ---: | --- |
| 1,000 | コイン 300 |
| 3,000 | 育成素材 小 5 |
| 5,000 | 通常ガチャチケット 1 |
| 8,000 | 育成素材 中 3 |
| 10,000 | 通常ガチャチケット 1 + 営業ブースト 1 |

週間報酬案:

| 条件 | 報酬 |
| --- | --- |
| 7日中5日ログイン | 通常ガチャチケット 2 |
| 週間35,000歩 | 通常ガチャチケット 3 |
| 週間50,000歩 | 10連チケット 1 |

報酬上限:

- MVPでは主要報酬対象を1日10,000歩までにする。
- 10,000歩を超えた分は累計歩数・称号・実績には反映するが、経済報酬は増やしすぎない。

## 6. ガチャ仕様

### ガチャ対象

MVPではスタッフガチャを中心にする。レシピと設備は初期解放、店舗ランク、クエスト報酬でも入手できるようにする。

ガチャ種別:

- 通常スタッフガチャ
- 将来: ピックアップスタッフガチャ
- 将来: レシピ・設備混合ガチャ

### レアリティ

| レアリティ | 排出率 | 役割 |
| --- | ---: | --- |
| N | 50% | 序盤の穴埋め、限界突破素材 |
| R | 35% | 実用的な基本スタッフ |
| SR | 12% | 編成軸になるスタッフ |
| SSR | 3% | 強力な個性と見た目の目玉 |

### 重複入手

- 同じスタッフを引いた場合は「覚醒ピース」に変換する。
- 覚醒ピースで最大レベル上限や固有効果を強化する。
- MVPでは覚醒段階は 0 から 3 まで。

## 10. 編成と営業計算

編成枠:

- スタッフ: 3枠
- レシピ: 3枠
- 設備: 3枠

営業は自動計算。プレイヤーは営業中に細かい操作をしない。

基本計算案:

```text
来客数 = 基礎来客数 × 店舗ランク補正 × 設備補正 × スタッフ補正 × ブースト補正
客単価 = 編成レシピ平均単価 × スタッフ補正 × 設備補正
売上 = 来客数 × 客単価
評判 = 来客数 × 満足度補正
```

放置営業:

- アプリを閉じていても一定時間まで売上が貯まる。
- 初期上限は2時間。
- 店舗ランクや設備で上限を伸ばす。

営業ブースト:

- 歩数報酬で入手。
- 1個使うと次の営業収益 +50%。
- MVPでは手動使用。

## 14. iOS実装方針

- SwiftUIで画面を構築する。
- HealthKitで歩数取得する。
- 開発初期はHealthKitが使えない環境のため、疑似歩数入力を残す。
- データモデルは `GameState`, `Staff`, `Recipe`, `Equipment`, `GachaService`, `StepRewardService`, `IdleEarningsService` に分ける。
- まずは単一ターゲットのiPhone縦画面固定で進める。
- UIはカフェ経営ゲームとして、実用的な情報密度とかわいさのバランスを取る。
